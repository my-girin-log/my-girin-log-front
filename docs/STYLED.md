# 내가그린기린기록 디자인 시스템 가이드

## 1. 디자인 철학 (Design Philosophy)
- **Neo-Brutalism (네오 브루탈리즘)**: 모든 UI 요소의 경계를 명확하게 나누며, 장식적인 그림자 블러(Blur)나 그라데이션을 배제합니다. 굵은 테두리와 단색 하드 섀도우를 통해 날것의 느낌과 현대적인 사용성을 동시에 제공합니다.
- **Developer / Terminal Aesthetic (개발자/터미널 감성)**: 타자기 스타일의 Monospace 폰트, 대괄호(`[ ]`)를 활용한 라벨링, 점선/실선 디바이더, 코드 에디터를 연상시키는 컴포넌트를 사용하여 주 타겟층(개발자)에게 친숙한 경험을 줍니다.
- **Retro Gamification (레트로 픽셀 게임)**: 도트(Pixel) 아트워크와 체력 게이지, 경험치(Lv) 같은 다마고치 스타일의 요소를 결합하여 '성장'이라는 키워드를 시각적으로 풀어냅니다.

## 2. 컬러 시스템 (Color System)

### 2.1. Brand & Accent Colors
- **Mint Green (`#AEE6C5` 계열)**: 브랜드 키(Key) 컬러. 활성화된 메뉴, 사용자(Me)의 채팅 말풍선, `Follow` 버튼 등 긍정적인 상호작용과 선택된 상태를 나타냅니다.
- **Forest Green (`#285B43` 계열)**: 강조가 필요한 주요 액션 버튼(`+ 새 회고 작성`, `SEND`, `공유`), 그리고 잔디(Activity Map) 그래프의 가장 높은 기여도 단계에 사용됩니다.

### 2.2. Base & Background Colors
- **Light Lavender/Blue-Grey (`#F0F2F8` 계열)**: 애플리케이션의 메인 배경색 및 비활성화된 캘린더 영역 등에 사용됩니다. 순백색의 컨텐츠 카드들이 돋보이도록 돕습니다.
- **White (`#FFFFFF`)**: 보관함 카드, 캘린더의 일자 박스, 시스템(Woody) 말풍선 등 사용자가 정보를 읽어야 하는 주 영역의 배경색입니다.

### 2.3. Stroke & Shadow (테두리 및 그림자)
- **Solid Dark (`#1E2128` 계열 / 거의 검은색에 가까운 다크 그레이)**: 앱 내의 모든 테두리(2px~3px)와 하드 섀도우, 주요 본문 텍스트에 사용되어 명도 대비를 극대화합니다.

### 2.4. Semantic / Tag Colors
- 카테고리나 상태를 나타내는 태그(Tag) 배경색으로 채도가 낮고 밝은 파스텔 톤(예: 연보라 `#E2E2F6`, 흰색, 연녹색 등)을 혼용하여 구분감을 줍니다.

## 3. 타이포그래피 (Typography)

### 3.1. Sans-serif (본문 및 주요 UI)
- **용도**: 페이지 제목(`보관함`, `홈`), 본문 텍스트, 버튼 라벨, 회고록 요약 등.
- **특징**: 가독성이 높은 고딕 계열(예: Pretendard)을 사용하며, 제목은 `Extra Bold` 이상으로 묵직하게, 본문은 `Regular`/`Medium`으로 깔끔하게 처리합니다.

### 3.2. Monospace (터미널/레트로 텍스트)
- **용도**: 레벨 및 타이틀(`Lv. 15 Growth Enthusiast`), 날짜 및 시간(`2023.10.27`, `10:45 AM`), 작성자 태그(`[SYSTEM]`, `[USER]`), 통계 수치(`365`, `1,204`), 상태 표시(`CONNECTION: SECURE`).
- **특징**: 자간이 일정한 폰트(예: Fira Code, Roboto Mono)를 사용하여 코드 데이터나 시스템 로그를 보는 듯한 느낌을 연출합니다. 대문자(Uppercase)와 혼용하여 레트로한 분위기를 강조합니다.

## 4. UI 컴포넌트 상세 스펙 (Component Specs)

### 4.1. 형태와 질감 (Shape & Texture)
- **Borders**: 모든 카드, 버튼, 입력창, 태그, 아바타 이미지는 `2px` 두께의 짙은 단색 테두리를 가집니다.
- **Shadows**: 입체감을 주는 요소(버튼, 말풍선, 리스트 카드 등)는 우측 하단 방향으로 번짐(Blur)이 `0px`인 오프셋 그림자(예: `box-shadow: 2px 2px 0px #1E2128;`)를 가집니다.
- **Border Radius**: 완전히 각진 모서리(`0px`) 또는 매우 미세하게 둥근 모서리(`2px~4px`)를 사용하여 날카롭고 단단한 블록 형태를 유지합니다.
- **Background Pattern**: 채팅룸(데일리 기록룸) 등 특정 화면의 배경에는 옅은 **도트 그리드(Dot Grid)** 패턴을 깔아 모눈종이나 설계 도면 같은 질감을 부여합니다.

### 4.2. 버튼 (Buttons)
- **Primary Action (다크 그린)**: 흰색 텍스트. 클릭 시 하드 섀도우가 사라지며 위치가 이동하는 형태의 프레스(Press) 인터랙션이 어울립니다.
- **Secondary Action (민트/화이트)**: 검은색 텍스트. 필터 버튼이나 서브 액션(`수정`)에 사용됩니다.

### 4.3. 말풍선 (Chat Bubbles)
- **꼬리 (Tail)**: 둥글지 않고 직각 형태로 꺾인 블록형 꼬리를 사용합니다.
- **디바이더 (Divider)**: 말풍선 내부에서 질문과 선택지 버튼 등을 나눌 때 실선 대신 **점선(`dashed`)** 테두리를 사용합니다.
- **발화자 식별**: 좌측 상단이나 우측 상단에 겹치도록 Monospace 폰트로 발화자(`Me [USER]`, `Woody [SYSTEM]`)를 라벨링합니다.
